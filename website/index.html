Voice-Enabled Espresso Maker

ECE 5725 Final Project - Scott Wu, Charles Bai

Overview

Aim to build a voice-controlled coffee machine "Aron" which is able to make coffee based on real time voice input. Raspberry Pi and Nespresso Mini Coffee Machine are the main hardware parts of the system. Snowboy is the main software for hotword detection of the system.

Hardware Design

Raspberry Pi & Circuits

We use a Raspberry Pi connected to a piTFT display touch screen. Raspberry Pi is perfect hardware system to run Snowboy hotword detection. Since we are using Raspberry Pi in command line mode, we use piTFT to display the command.

Coffee Machine & Add-on

The machine we are targeting is the Nespresso Essenza Mini. Although this is our ideal device, we would like our project to work on similar devices as well. We print an add-on holder that fits on top of the Espresso Maker that can press the buttons. This add-on fits snugly over the top of the machine, and comes with the appropriate mechanisms to use the device.

Servo

We connect the servo to a bread board and connect Raspberry Pi to the bread board using a breakout cable to control the servo. The servo is glued to the bottom of the Add-on to press the buttons on the coffee machine.

Software Design

Snowboy

We use Snowboy Hotword Detection powered by Kitt.ai to implement hotword recognition. Hotword recognition differs slightly from speech recognition because the recognizer must constantly be listening for the hotword. The Snowboy Hotword Detection framework allows us to recognize hotwords on the Raspberry Pi - no internet connection necessary. Since their software is proprietary, we must train our own hotwords on their website online.

Actuation

All our actuators are binary actuators (must be in one of two states), which means other than firing the actuators in the correct order, we do not need any precise timing, since the Raspberry Pi offers hardware PWM. Therefore, we use Python and the standard RPi.GPIO library to perform actuation of the device. We implement some error checking and safety features in this program to protect the servo.

Communication

The Hotword detection program and Actuation program communicate through a FIFO queue. As in Lab 3, we have a script to setup and start the two programs in the background. This script executes on boot.
